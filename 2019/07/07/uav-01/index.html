从头搭建基于ROS和px4的自主无人机开发环境

发表于 2019-07-07 更新于 2019-12-12

0. 引言

从今年四月底开始在罗老师门下学习人工智能的项目开发，“基于目标识别的自主无人机”是我参与的第一个项目。在开发环境的搭建中踩了很多的坑，中间也重装了很多次系统，在这过程中深觉记录研究成果的必要，罗老师也建议我对自己所作的工作进行记录，因此有了这个博客。

经过摸索，需要安装的开发环境主要包括：

1.Ubuntu 16.04 LTS 操作系统
2.Nvidia显卡驱动
3.CUDA + CUDNN 深度学习的加速引擎
4.px4固件（在安装过程中会安装opencv等依赖）
5.gazebo7仿真软件（经过测试，独立安装的gazebo7性能要远远优于px4或ROS整合安装的版本）
6.ROS kinetic

1. 安装UBUNTU16.04 LTS双系统

我原来使用的操作系统是win10, 但是px4等项目对windows的支持都十分的不友好，而且许多开源项目的开发平台都是Linux，所以安装Linux十分有必要。

在Linux的多个版本中，Ubuntu支持图形界面，是和windows用户操作习惯最贴近的，十分适合初学者入门，且行业内使用者众多，16.04LTS版本是一个成熟的长期支持版本，很多开源项目都是在该平台上开发和测试;相比之下，18.04LTS的版本支持仍然有一些问题，所以我最终决定安装UBUNTU16.04LTS双系统。

1.2 U盘安装Ubuntu

我们一般采用U盘启动的方式安装Ubuntu。事先在官网下载好Ubuntu 16.04LTS的镜像文件，再下载一个绿色的rufus（很小，一般只有几MB）。

插入U盘，打开rufus，选择烧写的盘和镜像，其它的默认就好。

烧写完毕后重新启动电脑，因为默认启动顺序是 windows > u盘启动，所以你还需要更改启动顺序。

更改启动顺序这一操作如何执行取决于你的电脑的BIOS，一般都是连续敲击F9～F12中的某一特定按键，不同品牌的电脑敲击的按键不同，这个需要自己网上查阅一下资料。我的电脑是惠普的，所以需要在重新启动后，电脑仍然黑屏时连续敲击F9，屏幕左下角会显示“F9 更改启动顺序的字样”。

在更改启动顺序的界面，按方向键选中U盘启动，按下回车键，就会进入Ubuntu安装。

后面的安装过程比较简单，可以参照网上其他的教程，但是有个比较大的坑需要注意一下，那就是系统的磁盘分区。

首先不要选择“ubuntu与windows共存的选项”，选择最下面的自定义，可以进入磁盘分区。

一般的分区方案是：

  1. 挂载点/；主分区；安装系统和软件；大小为30G；分区格式为ext4；
  2. 挂载点/home；逻辑分区；相当于“我的文档”；大小为硬盘剩下的; 分区格式ext4；
  3. swap；逻辑分区；充当虚拟内存；大小等于内存大小（本人16G）；分区格式为swap
  4. /boot ；主分区；逻辑分区； 大小为500M ；分区格式为ext4；

3. 安装Cuda和CUDNN

Cuda和CUDNN是Nvidia公司开发的GPU加速工具链，是所有基于Nvidia显卡加速的软件的基础。

3.1 命令行界面安装Cuda

首先在Nvidia官网下载cuda，需要注册并登录。cuda的版本不建议选择最新的，因为很多依赖都不支持，后续会出现很多问题，而cuda又是我们的地基，一旦出现问题一般都是重新安装系统的结果QAQ。（最初安装了cuda10.1版本，最后重装系统安装了cuda10.0）

将cuda的安装包放在～目录下（一般有2GB左右）。

Cuda的安装和Nvidia驱动类似，都需要关闭X服务，进入命令行界面，我们在命令行界面下进行安装，这里只列出代码，一些细节请看2.2。

这里有一个大坑：cuda的安装程序会询问你是否安装显卡驱动，因为我们之前已经装好了，所以填写n。（根据我的实测，如果没有安装驱动，使用cuda来安装显卡驱动，最后显卡会不可用，问题最终解决不了，只好重装系统QAQ；如果安装了显卡驱动，这里又选择了安装驱动，也会有兼容问题，最后结果还是重装了系统QAQ）所以，千万不要安装cuda自带的显卡驱动！！！

4. 搭建px4飞控固件的编译环境（基于官网脚本）

px4是目前主流的飞控固件之一，因为其开源性和良好的可拓展性而受到业内人士的青睐。px4的OFFBARD模式和mavlink协议允许我们用机载计算机对无人机进行控制。话不多说，我们开始对px4编译环境的搭建。

我第一次安装px4的时候，官网上还没有提供安装脚本，中途绕了很多弯路，遇到了很多问题。其中最大的就是python不兼容的问题，因为我安装了Anaconda的python3，而px4编译工具默认采用的是python2。这个问题困扰了我很久，最终发现最好的方法就是，在安装玩px4之后再安装python3。
